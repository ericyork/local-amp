How To Configure MAMP

Getting Started

MAMP is a bundle of software (including Apache, MySQL, and PHP) that allows you to create a local development environment. That is, it turns your desktop into development server.

This guide will walk you through the process of installing MAMP and configuring it enough to display a simple website.

1. Install MAMP

Visit this page and download the package for your OS. Then, double-click the .pkg file and follow the prompts for installation.

Once that's finished, start the MAMP app by double-clicking its icon.

2. Start the Servers and Load a Page

Click the "Start Servers" button on the MAMP dashboard and then click the 'Open Webstart Page' button. This should open localhost:8888/MAMP in your web browser, displaying a welcome page and useful links. You'll notice the lights on the MAMP dashboard for Apache and MySQL servers have changed to green.

Note: Once MAMP is installed, it creates a webroot directory at /Applications/MAMP/htdocs. Any files you put in this folder will be available to your browser at localhost:8888

3. Locate Important Files

The apache server configuration files are located at MAMP/conf/apache/httpd.conf/. These contain all the directives Apache uses to run the server.

Malicious PHP scripts are often used by hackers to compromise a server. You can harden your PHP settings by configuring the PHP Initialization file located at MAMP/conf/your-php-version/php.ini.

You can Google "Harden PHP" to find more information and to learn what settings to change.

4. Create a MySQL Database

Databases and users will be controlled using phpMyAdmin. Visit localhost:8888/phpMyAdmin in your browser.

Click on "Databases" in the nav menu and then type in a database name and select "utf8_general_ci" as the collation type.

You should see that a database with no tables was created in the side-panel.

5. Create a Database User

Click on the server link to go back, and then click on "User Accounts in the nav menu".

Under the "New" heading, click "Add user account." Then, type in a username and a passsword (twice). Leave everything else unchecked.

At the bottom of the page, click "Go" to submit the commands.

6. Give the User Privileges

From the "User Accounts" section, click on the "Edit Privileges" link next to the user whose permissions you want to edit.

In the sub-nav below the nav menu, click "Database". Then, select the database you just created and click "Go".

In the "Database-Specific Privileges", check all and click "Go".

Note: When you click the "Go" button, you'll notice a small, color-coded readout appears at the top of the page. This readout contains a status message as well as the complete SQL commands issued by your actions.

7. Add Files to the Webroot

Create an index.html file containing "Hello, World" to test browser display and add a CSS call to test that your files paths are working. Put both files in the /htdocs directory.

Your index file should contain the following code:

<!DOCTYPE html>
<html lang="en">
 <head>
  <meta charset="utf-8">
  <link rel="stylesheet" href="style.css">
 </head>
 <body>
  <h1>Hello, World!</h1>
 </body>
</html>

8. Secure the Files

Open up your command line and navigate to the root directory. On my local machine, the command looks like this (but yours may differ):

$ cd ../../Applications/MAMP/htdocs

Now, view the permissions of the files you just created.

$ ls -al

Notice your files have the permissions of 644. Go ahead and load up the page in your browser (by visiting localhosst:8888). Now, remove all permissions from the index file.

$ sudo chmod 000 index.html

Try to reload the page in your browser. What happened? You better change it back to 644!
